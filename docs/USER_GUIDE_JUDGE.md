2. Click "Login"

## Judge Dashboard Overview

The Judge Dashboard has four main sections:

- **Dashboard**: Overview with key metrics and recent cases
- **All Cases**: Complete case list with filtering and sorting
- **Precedents**: Search for relevant legal precedents
- **Analytics**: Case statistics and trends

## Dashboard Features

### Key Metrics

View system-wide statistics:
- **Total Cases**: All cases in the system
- **Pending Review**: Cases awaiting judicial action
- **Completed**: Cases that have progressed to hearing stage

### Recent Cases

See the 5 most recent cases with:
- Case title
- Filing citizen
- Status and priority badges
- Filing date

### Quick Actions

- **View All Cases**: Jump to the complete case list
- **Search Precedents**: Access the precedent search tool

## Managing Cases

### Viewing All Cases

1. Click "All Cases" in the sidebar
2. You'll see a table with all cases in the system

**Table Columns**:
- Case ID (shortened)
- Title
- Status badge
- Priority badge
- Filed date
- Actions (Generate Summary button)

### Filtering Cases

- **Search**: Find cases by title or case ID
- **Status Filter**: Filter by FILED, IN_REVIEW, or HEARING_SCHEDULED
- **Sort Options**: Sort by date, priority, or status

### Generating AI Summaries

1. Find the case in the case list
2. Click the "Generate" button in the Actions column
3. Wait for the AI to process (shows loading spinner)
4. Button changes to "Summary Done" when complete
5. View the summary by clicking on the case

**Note**: Each case can only have one summary generated. The button is disabled after generation.

### Case Details

Click on any case row to view:
- Full case description
- AI-generated summary (if available)
- Case metadata (ID, status, priority, dates)
- Filing citizen information

## Precedent Search

### How to Search

1. Click "Precedents" in the sidebar
2. Enter your search query
   - Example: "property boundary dispute"
   - Example: "contract breach"
3. (Optional) Select case type filter
4. (Optional) Set relevance threshold
5. Click "Search Precedents"

### Understanding Results

Results are displayed with:
- **Case Title**: Name of the precedent case
- **Summary**: Brief description of the case
- **Relevance Score**: Ranked from 1-5 (5 being most relevant)

### Sorting Results

- **Sort by Relevance**: Most relevant cases first (default)
- **Sort by Name**: Alphabetical order

### Search Tips

- Use specific keywords related to your case
- Try different phrasings if you don't find results
- Filter by case type to narrow results
- Results are ranked automatically by relevance

## Case Status Management

### Status Workflow

Cases progress through these statuses:
1. **FILED**: Initial submission by citizen
2. **IN_REVIEW**: Under judicial review
3. **HEARING_SCHEDULED**: Progressed to hearing stage

### Updating Case Status

1. View the case details
2. Select new status from dropdown
3. Click "Update Status"
4. Case is updated and citizen is notified

**Note**: Only judges can update case status

## Understanding Priority Levels

### HIGH Priority
Automatically assigned to cases containing urgent keywords:
- "urgent"
- "emergency"
- "immediate"
- "critical"
- "assault"

These cases require immediate attention.

### REGULAR Priority
Standard cases following normal processing timeline.

## Troubleshooting

### Can't See Cases?
- Refresh the page
- Verify you're logged in as a judge
- Check your internet connection

### Summary Generation Failed?
- Ensure the case has a description
- Try again after a few moments
- Contact support if issue persists

### Precedent Search Not Working?
- Check your search query
- Try broader keywords
- Verify you're connected to the internet
