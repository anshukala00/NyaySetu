# Alembic Database Migrations

This directory contains Alembic database migration scripts for the Nyaysetu Case Management System.

## Configuration

Alembic is configured to:
- Read `DATABASE_URL` from the `.env` file in the backend directory
- Auto-detect model changes from `app.database.Base` metadata
- Support autogenerate for creating migrations automatically

## Prerequisites

Before running migrations, ensure:
1. PostgreSQL is installed and running
2. Database is created (see `scripts/setup_database.bat`)
3. `.env` file is configured with correct `DATABASE_URL`
4. All models are imported in `alembic/env.py`

## Common Commands

### Check current migration version
```bash
alembic current
```

### Check if database is in sync with models
```bash
alembic check
```

### Create a new migration (autogenerate)
```bash
alembic revision --autogenerate -m "Description of changes"
```

### Create a new migration (manual)
```bash
alembic revision -m "Description of changes"
```

### Apply all pending migrations
```bash
alembic upgrade head
```

### Rollback one migration
```bash
alembic downgrade -1
```

### Rollback to specific revision
```bash
alembic downgrade <revision_id>
```

### View migration history
```bash
alembic history
```

### View SQL without executing
```bash
alembic upgrade head --sql
```

## Migration Workflow

### 1. Create Models
Define your SQLAlchemy models in the `models/` directory:

```python
from app.database import Base
from sqlalchemy import Column, String
from sqlalchemy.dialects.postgresql import UUID
import uuid

class User(Base):
    __tablename__ = "users"
    
    id = Column(UUID(as_uuid=True), primary_key=True, default=uuid.uuid4)
    email = Column(String, unique=True, nullable=False)
    # ... other fields
```

### 2. Import Models in env.py
Add model imports to `alembic/env.py` so Alembic can detect them:

```python
# Import all models here so Alembic can detect them for autogenerate
from models.user import User
from models.case import Case
```

### 3. Generate Migration
Run autogenerate to create migration script:

```bash
alembic revision --autogenerate -m "Create users and cases tables"
```

### 4. Review Migration
Check the generated migration file in `alembic/versions/` and verify:
- All expected changes are included
- No unintended changes are present
- Upgrade and downgrade functions are correct

### 5. Apply Migration
Apply the migration to the database:

```bash
alembic upgrade head
```

### 6. Verify
Check that tables were created correctly:

```bash
alembic current
```

## Best Practices

1. **Always review autogenerated migrations** - Alembic may not detect all changes correctly
2. **Test migrations on development database first** - Never run untested migrations on production
3. **Write reversible migrations** - Ensure downgrade() function properly reverses upgrade()
4. **Use descriptive migration messages** - Makes it easier to understand migration history
5. **Commit migrations to version control** - Migration files should be tracked in git
6. **Never edit applied migrations** - Create a new migration to fix issues
7. **Import all models in env.py** - Required for autogenerate to detect changes

## Troubleshooting

### "No module named 'app'" error
- Ensure you're running alembic from the `backend/` directory
- Check that `sys.path` is correctly configured in `env.py`

### Autogenerate not detecting changes
- Verify models are imported in `env.py`
- Check that models inherit from `Base`
- Ensure `target_metadata = Base.metadata` is set in `env.py`

### Database connection errors
- Verify `DATABASE_URL` in `.env` file
- Check PostgreSQL is running
- Verify database exists and credentials are correct

### Migration conflicts
- Use `alembic history` to view migration tree
- Resolve conflicts by creating merge migrations if needed
